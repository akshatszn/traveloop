The problem

Travel planning is still a messy stack of tabs, chats, maps, notes, and guesses.

People do not just need a list of places. They need a realistic, shared operating plan that handles time, budget, geography, interests, and group decisions.

01

Planning is fragmented

Inspiration lives on TikTok and blogs, logistics live in Google Maps, decisions live in group chat, and the actual plan often lives nowhere.

02

AI suggestions are not operational

Generic itineraries ignore budget pressure, transit buffers, daily pacing, and collaboration.

03

Group decisions create drag

People need a shared source of truth that feels as easy as Notion, as polished as Airbnb, and as structured as Linear.

Opportunity

There is room for a planning layer between inspiration and booking.

Traveloop owns the high-intent moment where travelers know they are going somewhere, but still need to turn possibilities into a plan everyone trusts.

Product

An AI travel workspace that creates the itinerary, then keeps it editable.

Users enter destinations, dates, budget, travel style, and interests. Traveloop returns a complete multi-city plan with day blocks, costs, activity recommendations, notes, packing, and share pages.

  • AI-assisted trip creation wizard
  • Drag-and-drop timeline with sticky day navigation
  • Budget estimation and activity cost visibility
  • Public shareable itinerary pages
  • Premium dashboard for recent trips and analytics

Architecture

Production-shaped monorepo, ready for Vercel and Railway.

The build is separated cleanly: Next.js web on Vercel, Express API on Railway, and PostgreSQL through Neon or Railway.

Web

Next.js 15 App Router

  • TypeScript and TailwindCSS
  • Framer Motion interactions
  • Zustand auth and wizard state
  • React Query API cache
API

Express + Prisma

  • JWT authentication
  • Bcrypt password hashing
  • Zod validation
  • Service-layer architecture
Data

PostgreSQL schema

  • Users, trips, stops, cities
  • Activities and itinerary items
  • Budgets, notes, packing
  • Public share slugs
